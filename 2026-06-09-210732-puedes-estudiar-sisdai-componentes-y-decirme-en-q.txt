⏺ sisdai-componentes es una biblioteca de componentes reutilizables para Vue.js,
  diseñada específicamente para alinearse con el Sistema de Diseño y 
  Accesibilidad para la Investigación (Sisdai).

  En esencia, consiste en un conjunto de piezas de interfaz de usuario (UI)
  preconstruidas que permiten a los desarrolladores crear sitios web que sean
  consistentes, accesibles y alineados con la identidad institucional del
  Gobierno de México, particularmente para proyectos coordinados por el Conahcyt
  y desarrollados por CentroGeo.

  Aquí tienes los puntos clave de lo que representa:

  1. Objetivos Principales

  - Homologación Visual: Asegura que todos los sitios que usen la biblioteca
  sigan los mismos estilos y guías visuales del sistema Sisdai.
  - Accesibilidad: Implementa HTML semántico y reglas de accesibilidad para que
  los sitios sean utilizables por la mayor cantidad de personas posible.
  - Eficiencia de Desarrollo: Reduce la cantidad de código repetitivo al
  proporcionar componentes comunes (barras de navegación, menús, formularios) ya
  listos para usar.
  - Facilidad de Actualización: Permite actualizar elementos institucionales
  (como pies de página o cabeceras) de manera global y sencilla.

  2. Componentes Incluidos

  La biblioteca ofrece una amplia gama de componentes, desde elementos básicos
  de formulario hasta estructuras complejas de navegación:
  - Formularios: SisdaiAreaTexto, SisdaiBotonRadio, SisdaiCasillaVerificacion,
  SisdaiSelector.
  - Navegación e Identidad: SisdaiNavegacionGobMx, SisdaiNavegacionPrincipal,
  SisdaiPiePaginaGobMx, SisdaiPiePaginaConahcyt.
  - Interactividad y Layout: SisdaiModal, SisdaiPestanias, SisdaiMenuLateral,
  SisdaiBotonFlotante.
  - Utilidades: SisdaiAudio, SisdaiImagenPngWebp, SisdaiInfoDeDespliegue.

  3. Aspectos Técnicos

  - Framework: Basado en Vue.js utilizando la Composition API.
  - Estilos: Utiliza CSS puro (evitando preprocesadores como SCSS) para mantener
  la simplicidad y compatibilidad.
  - Documentación: Cuenta con un sitio de documentación generado con VuePress.
  - Licencia: Es software libre bajo la licencia LGPLv3, promoviendo la
  soberanía tecnológica y los estándares abiertos.
  - Distribución: Se distribuye a través de npm
  (@centrogeomx/sisdai-componentes) y GitHub.

  En resumen, es el "kit de construcción" oficial para asegurar que las
  aplicaciones de investigación y gubernamentales asociadas a Sisdai sean
  visualmente coherentes, accesibles y fáciles de mantener.

⏺ He realizado los siguientes cambios para crear la versión en JavaScript puro
  del componente SisdaiAreaTexto y utilizarla en el ejemplo de la documentación:

  1. Creación del componente JS: He creado el archivo
  src/componentes/area-texto/SisdaiAreaTextoJS.js. Este archivo implementa el
  componente como un Web Component (Custom Element) llamado
  <sisdai-area-texto-js>. Utiliza el DOM ligero (Light DOM) para heredar los
  estilos globales del proyecto y replica toda la lógica de accesibilidad y
  estructura del componente original en Vue.
  2. Exportación a la raíz: He actualizado la cadena de exportaciones para que
  el nuevo componente esté disponible desde el punto de entrada principal de la
  biblioteca:
    - src/componentes/area-texto/index.js $\rightarrow$ src/componentes/index.js
  $\rightarrow$ src/index.js.
  3. Actualización del ejemplo: He modificado
  docs/.vitepress/components/area-texto/basico.vue para utilizar la versión en
  JavaScript.

  Nota sobre el uso en Vue (v-model):
  Como se discutió, al tratarse de un Web Component, el v-model de Vue no
  funciona de la misma manera que en un componente .vue. Por ello, en el archivo
  basico.vue he sustituido v-model="areaEjemplo" por la combinación:
  - :model-value="areaEjemplo": Para pasar el valor al componente.
  - @input="areaEjemplo = $event.detail": Para capturar los cambios, ya que el
  componente JS emite un evento personalizado con el valor en el campo detail.

  Ahora el ejemplo básico de la documentación utiliza la implementación pura de
  JavaScript.

⏺ El error ocurría debido a un problema común en los Web Components: un bucle de
  renderizado.

  Cuando escribías en el área de texto, el evento input actualizaba el atributo
  model-value mediante setAttribute. Esto disparaba el método
  attributeChangedCallback, el cual llamaba a render(). Al ejecutar render(), el
  componente borraba todo su contenido interno (this.innerHTML = ...) y lo
  volvía a crear. En ese instante, el elemento <textarea> donde estabas
  escribiendo desaparecía y era reemplazado por uno nuevo, provocando que
  perdieras el foco inmediatamente después de escribir el primer carácter.

  He solucionado esto realizando el siguiente ajuste:

  He modificado el attributeChangedCallback para que detecte si el atributo que
  cambió es model-value.
  - Si es model-value, ahora el componente simplemente busca el <textarea>
  existente y actualiza su valor directamente, sin volver a renderizar todo el 
  HTML.
  - Si cambia cualquier otro atributo (como la etiqueta o la obligatoriedad),
  entonces sí se ejecuta el render() completo para reflejar los cambios
  estructurales.

  Con esto, el foco se mantiene en el campo de texto y ya deberías poder
  escribir normalmente en el ejemplo de la documentación.
